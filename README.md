## AIDEA OS 制作メモ

### 制作意図

中小企業向けの「AI秘書・業務自動化サービス」を、安価なSaaSテンプレートではなく、高級B2B SaaS / AI導入支援会社の実績サンプルとして見える品質で設計しています。AIそのものを強調するのではなく、問い合わせ、日程調整、社内ナレッジ検索、下書き作成、レポート化が業務フローとして整う印象を重視しました。

### デザイン方針

- 濃紺、白、深いティール、控えめなグレーを中心にした落ち着いたB2B配色
- ロボット画像や過剰な吹き出し表現は使わず、管理画面と業務ボードを主役にする
- セクション数を絞り、余白を広く取り、1つずつ大きく見せる
- Hero右側はスマホ画面ではなく、実際のSaaS管理画面に近い業務ダッシュボードUI
- コピーは誇張を避け、「人が判断し、AIが下準備する」導入しやすい表現に統一

### 使用技術

- HTML / CSS / JavaScript
- 既存の静的サイト構成に合わせ、ビルド不要で動作
- アイコンは既存読み込みのLucide CDNを使用
- チャットデモは `assets/app.js` の既存ローカル実装を利用

### セクション構成

- Header: Service / Workflow / Dashboard / Plan / Contact
- Hero: AIDEA OSの価値と業務ダッシュボードUI
- Problem: AIツール導入後に業務が変わらない理由
- Workflow: 受信からレポート化までの導入後フロー
- Dashboard: AIの裏側の処理を見える化する管理画面
- Implementation Design Board: 業務棚卸し、承認フロー、運用改善の設計ボード
- Value: 対応漏れ、属人化、判断時間に絞った提供価値
- Contact CTA: 1業務からの無料相談導線

### 画像 / アセットの扱い

AIDEA OSページでは、第三者写真やブランド素材を使わず、HTML/CSSで作成したオリジナルのUIモックをビジュアルの中心にしています。外部画像のライセンス依存を避け、ポートフォリオ掲載や商用提案時に説明しやすい構成です。既存のデモ一覧ページにはUnsplash画像の外部参照がありますが、AIDEA OS本体ページの主要ビジュアルには使用していません。

### 今後の改善案

- 実案件向けには問い合わせフォーム送信先、CRM連携、Slack通知の実装を追加
- UIモックを実データ風のスクリーンショット画像として書き出し、SNS掲載用サムネイルを作成
- Lighthouse計測に合わせてLucideのCDN読み込みをローカルSVGまたは最小アイコンに置き換える
- 導入事例、料金、FAQを追加して商談用ページとして拡張
